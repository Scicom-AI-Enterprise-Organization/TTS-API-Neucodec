# ---------------------------------------------------------------------------
# Dependencies to run the TTS/VC decode API on a Huawei Ascend NPU (910B3).
#
# Target box: aarch64, Python 3.9, CANN 8.5.2 (npu-smi/driver 25.5.1).
# All wheels install straight from PyPI (no special index). Before installing
# and before running, source the CANN env so torch_npu can load its backend:
#     source /usr/local/Ascend/ascend-toolkit/set_env.sh
#     python -m venv venv && source venv/bin/activate
#     pip install -U pip && pip install -r requirements-npu.txt
#
# This is the NPU counterpart of requirements.txt. Differences vs the CUDA reqs:
#   * torch/torchaudio pinned to 2.8.0 (the version torch_npu 2.8.0.post5 matches;
#     the CUDA reqs use 2.9.1, which torch_npu does not support yet).
#   * torch_npu added; provides the 'npu' device backend.
#   * NO torchtune / torchao: torchtune hard-imports torchao, which is broken on
#     Python 3.9 (py3.10 `X | Y` syntax) and also breaks transformers. NeuCodec
#     only needs torchtune's RotaryPositionalEmbeddings, which is vendored in
#     app/neucodec/_rope.py instead.
#   * The gradio / malaya / fasttext-language-detection / sentry / loki / wandb /
#     tensorboard extras are dropped: not needed for the decode+serving path, and
#     fasttext/librosa/loki/sentry are imported optionally in app/main.py.
# ---------------------------------------------------------------------------
